## 1. 功能需求

### 1.1 爬蟲失敗重試機制 [優先級: 最高]

#### 需求 R1.1.1 - 可配置重試上限
- **描述**: 爬蟲執行失敗時，自動重試指定次數
- **驗收標準**:
  - [ ] 支援可配置的最大重試次數（建議預設 3 次）
  - [ ] 用戶可在配置檔中設定重試上限
  - [ ] 達到上限後拋出明確的錯誤訊息

#### 需求 R1.1.2 - 智能退避策略
- **描述**: 重試間隔應逐次增加，避免頻繁請求
- **驗收標準**:
  - [ ] 實現指數退避 (exponential backoff) 機制
  - [ ] 支援自訂退避基數和最大等待時間
  - [ ] 可選的隨機抖動 (jitter) 防止雷鳴羊群

#### 需求 R1.1.3 - 錯誤分類
- **描述**: 區分瞬時錯誤和永久錯誤，只重試適當的錯誤類型
- **驗收標準**:
  - [ ] 定義瞬時錯誤清單（網路超時、500 服務器錯誤等）
  - [ ] 定義永久錯誤清單（404、403、無效參數等）
  - [ ] 永久錯誤立即失敗，不重試

#### 需求 R1.1.4 - 日誌記錄
- **描述**: 詳細記錄每次重試嘗試
- **驗收標準**:
  - [ ] 記錄每次嘗試的時間戳、錯誤原因、下次重試時間
  - [ ] 支援不同日誌級別（DEBUG、INFO、ERROR）
  - [ ] 最終失敗時生成摘要報告

### 1.2 MP3 轉換與合併 [優先級: 高]

#### 需求 R1.2.1 - MP3 格式轉換
- **描述**: 將抓取的音頻文件轉換為 MP3 格式
- **驗收標準**:
  - [ ] 支援常見輸入格式 (WAV, AAC, OGG, FLAC 等)
  - [ ] 可配置輸出比特率 (預設 128kbps)
  - [ ] 轉換過程不阻塞其他操作

#### 需求 R1.2.2 - 自動音頻合併
- **描述**: 根據指定時長，自動合併多個 MP3 檔案
- **驗收標準**:
  - [ ] 支援可配置的目標時長 (預設 11 小時)
  - [ ] 計算多個 MP3 的總時長
  - [ ] 自動分組，使每組接近目標時長（容差範圍 ±10%）
  - [ ] 保持文件順序和連接無縫

#### 需求 R1.2.3 - 時長計算與驗證
- **描述**: 準確計算音頻時長，驗證合併結果
- **驗收標準**:
  - [ ] 提取 MP3 元數據並計算總時長
  - [ ] 容差檢查：實際時長 vs 目標時長
  - [ ] 生成時長報告和分組摘要

### 1.3 MP4 最終轉換 [優先級: 中]

#### 需求 R1.3.1 - MP3 到 MP4 轉換
- **描述**: 將合併後的 MP3 轉換為 MP4 容器
- **驗收標準**:
  - [ ] 支援添加靜止視頻或純黑視頻背景
  - [ ] 保留原始音頻比特率
  - [ ] 可配置輸出解析度 (預設 1080p 或無視頻)

#### 需求 R1.3.2 - 元數據支援
- **描述**: 在 MP4 中嵌入音訊名稱、時長等元數據
- **驗收標準**:
  - [ ] 支援自訂標題、藝術家、相冊等 ID3 標籤
  - [ ] 編碼時長和其他媒體資訊

## 2. 非功能需求

### 性能
- 轉換過程應利用多核心（如適用）
- 對於大文件，應支援增量或流式處理
- 合併 10 小時以上的音頻應在 5 分鐘內完成

### 穩定性
- 失敗重試不應掩蓋真正的問題（錯誤分類準確度 > 95%）
- 轉換失敗時應提供清晰的錯誤訊息

### 配置與可維護性
- 所有可配置值應在 `.env` 或配置文件中定義
- 支援環境變數覆蓋預設配置
- 清晰的模塊化設計，便於未來擴展

## 附錄: 研究摘要

### MP3/MP4 轉換庫研究
- **FFmpeg**: 業界標準，功能完整，Bun 相容性好
- **fluent-ffmpeg**: Node.js 便利包裝，但在 Bun 中需驗證
- **音頻庫**: tone.js, wav.js 等適合輕量級處理

### 重試機制最佳實踐
- 指數退避是標準做法
- 錯誤分類是避免浪費的關鍵
- 熔斷器模式適合長期穩定性
