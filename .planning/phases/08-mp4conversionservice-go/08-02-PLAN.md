**Section 5: Troubleshooting** (5-7 common issues)

Issue 1: "MP4_GO_ENABLED=true but Go backend not used"
- Solution: Verify binary path exists and is executable: `ls -la <path>/kinetitext-mp4convert`
- Check logs for initialization errors

Issue 2: "Go backend fails, fallback to Bun works"
- Expected behavior: logging should show warning, conversion succeeds via Bun
- If persistent: check binary compatibility (Go 1.20+, FFmpeg 4.0+)

Issue 3: "Metadata not embedded in M4A file"
- Verify metadata fields are provided to convert() method
- Use ffprobe to inspect: `ffprobe -show_format <output.m4a>`
- Check logger for metadata serialization errors

Issue 4: "Timeout during conversion"
- Long audio files may exceed default 60s timeout
- Increase MP4_GO_TIMEOUT_MS env var (e.g., 120000 for 2 minutes)

Issue 5: "UTF-8 metadata corrupted (Chinese characters)"
- Go natively supports UTF-8; verify input strings are valid UTF-8
- Check terminal encoding: `echo $LANG`

Issue 6: "Binary not found on different system"
- Need to recompile Go backend: `cd kinetitext-go && make build-mp4convert`
- Or set MP4_GO_BINARY_PATH to absolute path

Issue 7: "Performance worse than expected"
- Verify Bun overhead: disable Go backend and measure baseline
- Check system load (FFmpeg is CPU-intensive)
- Refer PERF_REPORT.md methodology

**Section 7: FAQ** (6-8 questions)

Q: Will Go backend be used automatically?
A: Only if enabled via MP4_GO_ENABLED env var or config. Default is Bun FFmpeg (backward compatible).

Q: Can I switch between Bun and Go at runtime?
A: Go backend must be initialized once at startup via initGoBackend(). For runtime switching, create separate service instances.

Q: Is metadata embedding mandatory?
A: No, metadata is optional. Conversions without metadata work fine.

Q: What if kinetitext-go repo is not available?
A: Go backend will gracefully disable (log warning). Conversion continues via Bun FFmpeg.

Q: How do I know if Go backend is active?
A: Check logs for "Go backend initialized successfully" message. Alternatively, time conversions (Go is faster for long audio).

Q: Is there performance cost for metadata?
A: Minimal (~2-3ms). Metadata embedding does not significantly impact conversion time.

Q: Can I use Go backend in Windows?
A: Requires subprocess JSON IPC (not Bun FFI). Windows support planned for Phase 9.

Q: Next optimizations after Phase 8?
A: Phase 9 may consider ContentCleaner Go migration. See ROADMAP.md.
