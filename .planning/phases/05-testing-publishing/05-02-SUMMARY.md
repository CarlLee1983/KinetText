## Decisions Made

1. Each test suite uses its own `mkdtemp` call rather than sharing `e2eRootDir` — avoids beforeAll ordering issues when suites run in parallel.
2. Fixtures use lazy generation (files created inside the suite's beforeAll) rather than pre-committed sample files.
3. Merging E2E uses `targetSeconds=10, tolerancePercent=20` to create meaningful grouping with 3s test files while keeping total runtime under 2 minutes.
