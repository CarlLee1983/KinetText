  ## 目錄

  ## RetryService

  ### 用途
  自動重試失敗的操作，支援指數退避、錯誤分類、日誌記錄。

  ### 方法

  #### execute<T>(operation, options?): Promise<T>
  執行操作並在失敗時自動重試。

  **參數**:
  - `operation`: 要執行的異步函數
  - `options`: 可選配置 (maxRetries, baseDelay, maxDelay)

  **回傳**:
  - 成功: 操作的結果
  - 失敗: 拋出 Error

  **範例**:
  ```typescript
  const result = await retryService.execute(
    () => fetch("https://example.com"),
    { maxRetries: 3 }
  )
  ```

  ### 配置 (環境變數)
  - `RETRY_MAX_ATTEMPTS`: 最大重試次數 (預設 3)
  - `RETRY_BASE_DELAY_MS`: 基礎延遲 (毫秒, 預設 100)
  - `RETRY_MAX_DELAY_MS`: 最大延遲 (毫秒, 預設 10000)

  ---

  ## AudioConvertService

  ### 用途
  將音頻檔案轉換為 MP3 格式，支援 WAV, AAC, OGG, FLAC 等輸入格式。

  ### 方法

  #### convert(inputPath, outputPath, config?): Promise<AudioConvertResult>
  轉換單個音頻檔案。

  **參數**:
  - `inputPath`: 輸入檔案路徑 (必需)
  - `outputPath`: 輸出檔案路徑 (必需)
  - `config`: 可選配置 (bitrate, sampleRate, channels)

  **回傳**:
  ```typescript
  {
    success: boolean
    outputPath: string
    inputFormat?: string
    duration?: number // 秒
    fileSizeBytes?: number
    error?: string
  }
  ```

  **範例**:
  ```typescript
  const result = await audioConvertService.convert(
    "input.wav",
    "output.mp3",
    { bitrate: "192k" }
  )
  ```

  #### convertBatch(options): Promise<AudioConvertResult[]>
  轉換多個音頻檔案。

  **參數**:
  - `inputFiles`: 輸入檔案陣列
  - `outputDir`: 輸出目錄
  - `config`: 批量配置

  **回傳**: 結果陣列

  ### 配置
  - `AUDIO_BITRATE`: 比特率 (預設 128k, 支援 64k-320k)
  - `AUDIO_SAMPLE_RATE`: 採樣率 (預設 44100)
  - `AUDIO_CHANNELS`: 聲道數 (預設 2)

  ## AudioMergeService

  ### 用途
  合併多個 MP3 檔案，支援按時長或按數量分組。

  ### 方法

  #### mergeBatch(options): Promise<GroupingReport>
  根據配置合併和分組音頻檔案。

  **參數**:
  ```typescript
  {
    files: string[]               // 輸入檔案陣列
    targetDuration?: number       // 目標時長 (秒)
    outputDir: string             // 輸出目錄
    groupingStrategy?: 'duration' | 'count'
  }
  ```

  **回傳**:
  ```typescript
  {
    groups: GroupSummary[]
    totalDuration: number
    actualDuration: number
    oversizedSingleFile?: {
      filename: string
      duration: number
    }
  }

  interface GroupSummary {
    groupId: string
    fileCount: number
    estimatedDuration: number
    actualDuration: number
    mergeResult: {
      outputPaths: string[]
      totalDuration: number
    }
  }
  ```

  **範例**:
  ```typescript
  const report = await audioMergeService.mergeBatch({
    files: mp3Files,
    targetDuration: 39600, // 11 小時
    outputDir: "./merged"
  })
  ```

  計算音頻檔案的時長，精度 < 1%。

  ### 方法

  #### getDuration(filePath): Promise<number>
  取得檔案時長 (秒)。

  **參數**:
  - `filePath`: 音頻檔案路徑

  **回傳**: 時長 (秒)

  **範例**:

  ## 完整範例: 端到端工作流程

  ```typescript
  import { RetryService, AudioConvertService, AudioMergeService, MP4ConversionService, MP4Pipeline } from "./src/core"

  async function processAudioBook() {
    // 1. 初始化服務
    const retryService = new RetryService()
    const audioConvertService = new AudioConvertService(retryService)
    const durationService = new DurationService()
    const audioMergeService = new AudioMergeService(durationService)
    const mp4ConversionService = new MP4ConversionService(retryService)
    const mp4Pipeline = new MP4Pipeline(mp4ConversionService)

    // 2. Phase 2: 轉換 WAV → MP3
    const mp3Files = await audioConvertService.convertBatch({
      inputFiles: ["chapter1.wav", "chapter2.wav"],
      outputDir: "./mp3",
      config: { bitrate: "128k" }
    })

    // 3. Phase 3: 合併
    const mergeReport = await audioMergeService.mergeBatch({
      files: mp3Files.map(r => r.outputPath),
      targetDuration: 39600,
      outputDir: "./merged"
    })

    // 4. Phase 4: 轉換 MP3 → M4A
    const mp4Report = await mp4Pipeline.execute({
      inputDir: "./merged",
      outputDir: "./m4a",
      metadata: {
        "merged_001.mp3": {
          title: "第一部分",
          artist: "作者",
          album: "書名"
        }
      }
    })

    console.log(`完成! 轉換 ${mp4Report.fileCount} 個檔案`)
  }
  ```

  本文檔說明所有可配置的選項、環境變數及其預設值。

  2. 編輯 `.env` 設置您的偏好值

  ## 環境變數參考

  ### 重試機制 (Phase 1)

  | 變數名稱 | 預設值 | 範圍 | 說明 |
  |---------|--------|------|------|
  | `RETRY_MAX_ATTEMPTS` | 3 | 1-10 | 最大重試次數 |
  | `RETRY_BASE_DELAY_MS` | 100 | 1-1000 | 初始延遲 (毫秒) |
  | `RETRY_MAX_DELAY_MS` | 10000 | 1000-60000 | 最大延遲 (毫秒) |
  | `RETRY_BACKOFF_MULTIPLIER` | 2 | 1.5-3 | 指數退避倍數 |
  | `RETRY_JITTER_ENABLED` | true | true/false | 啟用隨機抖動 |

  **說明**:
  - 重試使用指數退避策略
  - 計算延遲公式: `min(baseDelay * (multiplier ^ attempt), maxDelay) + jitter`
  - 適用於所有服務的遠程操作

  **範例**:
  ```bash
  RETRY_MAX_ATTEMPTS=5
  RETRY_BASE_DELAY_MS=200
  RETRY_MAX_DELAY_MS=30000
  ```

  ### 音頻轉換 (Phase 2)

  | 變數名稱 | 預設值 | 範圍 | 說明 |
  |---------|--------|------|------|
  | `AUDIO_BITRATE` | 128k | 64k-320k | MP3 比特率 |
  | `AUDIO_SAMPLE_RATE` | 44100 | 8000-48000 | 採樣率 (Hz) |
  | `AUDIO_CHANNELS` | 2 | 1-2 | 聲道數 |
  | `AUDIO_FORMAT` | mp3 | mp3 | 輸出格式 |
  | `AUDIO_CONVERT_MAX_CONCURRENCY` | 2 | 1-8 | 並行轉換數 |

  **說明**:
  - 比特率越高，品質越好，檔案越大
  - 常見設置: 128k (質量), 192k (均衡), 256k (高品質)
  - 並行度取決於 CPU 核心數

  **範例**:
  ```bash
  AUDIO_BITRATE=192k
  AUDIO_SAMPLE_RATE=44100
  AUDIO_CHANNELS=2
  AUDIO_CONVERT_MAX_CONCURRENCY=4
  ```

  ### 音頻合併 (Phase 3)

  | 變數名稱 | 預設值 | 範圍 | 說明 |
  |---------|--------|------|------|
  | `AUDIO_MERGE_TARGET_DURATION` | 39600 | 1800-86400 | 目標時長 (秒) |
  | `AUDIO_MERGE_TOLERANCE_PERCENT` | 10 | 1-50 | 容差百分比 |
  | `AUDIO_MERGE_MAX_CONCURRENCY` | 2 | 1-8 | 並行合併數 |

  **說明**:
  - 39600 秒 = 11 小時 (預設值適合音頻書)
  - 容差 10% 意即 ±10% 的目標時長
  - 並行度影響記憶體使用

  **預設值組合**:
  - 音頻書: 39600 秒 (11 小時)
  - Podcast 集合: 7200 秒 (2 小時)
  - 講座: 3600 秒 (1 小時)

  **範例** (設置 8 小時):

  AUDIO_MERGE_TOLERANCE_PERCENT=15

  | 變數名稱 | 預設值 | 範圍 | 說明 |
  |---------|--------|------|------|
  | `MP4_BITRATE` | 256k | 96k-320k | AAC 比特率 |
  | `MP4_FORMAT` | m4a | m4a | 輸出格式 |
  | `MP4_MAX_CONCURRENCY` | 2 | 1-8 | 並行轉換數 |
  | `MP4_INCLUDE_METADATA` | true | true/false | 嵌入元資料 |

  **說明**:
  - M4A 使用 AAC 編碼，比 MP3 更高效
  - 256k AAC ≈ 192k MP3 品質
  - 元資料包括 title, artist, album

  **範例** (高品質設置):
  ```bash
  MP4_BITRATE=320k
  MP4_MAX_CONCURRENCY=4
  ```

  ## 1. 安裝與環境

  ### Q: "command not found: bun"
  **A**: Bun 未安裝或不在 PATH 中
  ```bash
  # 安裝 Bun
  curl -fsSL https://bun.sh/install | bash

  # 驗證安裝
  bun --version

  ### Q: "FFmpeg not found"
  **A**: 系統未安裝 FFmpeg
  ```bash
  # macOS
  brew install ffmpeg

  # Ubuntu/Debian
  sudo apt-get install ffmpeg

  # Windows
  choco install ffmpeg
  ```

  ### Q: "module not found"
  **A**: 缺少依賴包
  ```bash
  # 重新安裝依賴
  bun install

  # 清理快取後重試
  rm -rf node_modules bun.lock
  bun install
  ```

  ## 2. 重試與錯誤

  ### Q: 爬蟲反覆失敗
  **A**: 可能是網頁防護或網路問題
  ```bash
  # 增加重試次數
  RETRY_MAX_ATTEMPTS=5 bun run start "URL"

  # 增加重試延遲
  RETRY_BASE_DELAY_MS=500 bun run start "URL"

  # 檢查日誌
  bun run start "URL" 2>&1 | grep "retry"
  ```

  ### Q: "Error: Failed with exit code 254"
  **A**: FFmpeg 執行失敗，通常是輸入檔案問題
  ```bash
  # 檢查輸入檔案是否存在和有效
  file input.wav

  # 使用 ffprobe 驗證
  ffprobe input.wav

  # 如果無效，轉換格式後重試
  ffmpeg -i input.wav -c:a pcm_s16le -f wav output.wav
  ```

  ### Q: "Retries exhausted"
  **A**: 重試次數用盡仍未成功
  ```bash
  # 檢查網路連接
  ping example.com

  # 查看詳細錯誤日誌
  bun run start "URL" 2>&1 | grep -A 5 "exhausted"

  # 手動重試失敗章節
  bun run retry-failed "小說名稱"
  ```

  ## 3. 音頻轉換

  ### Q: "轉換超時"
  **A**: 檔案太大或系統繁忙
  ```bash
  # 減少並行轉換數
  AUDIO_CONVERT_MAX_CONCURRENCY=1 bun run audiobook "小說名稱"

  # 增加重試延遲
  RETRY_MAX_DELAY_MS=60000 bun run audiobook "小說名稱"
  ```

  ### Q: "輸出檔案損壞"
  **A**: FFmpeg 編碼錯誤
  ```bash
  # 驗證輸出
  ffprobe output.mp3

  # 使用不同的比特率重試
  AUDIO_BITRATE=192k bun run audiobook "小說名稱"

  # 檢查磁碟空間
  df -h
  ```

  ### Q: "支援不了的音頻格式"
  **A**: FFmpeg 不支援該格式
  ```bash
  # 查看支援的編碼
  ffmpeg -codecs | grep audio

  # 先轉換為支援的格式
  ffmpeg -i input.xyz -c:a pcm_s16le -f wav output.wav
  ```

  ## 4. 音頻合併

  ### Q: "時長計算不準確"
  **A**: 通常是音頻有特殊標記
  ```bash
  # 檢查元資料
  ffprobe -v quiet -print_format json -show_format input.mp3

  # 重新編碼檔案以移除特殊標記
  ffmpeg -i input.mp3 -c:a libmp3lame -b:a 192k clean.mp3
  ```

  ### Q: "無法分組到目標時長"
  **A**: 檔案數或容差設置不適合
  ```bash
  # 增加容差百分比
  bun run merge-mp3 --input=... --tolerance=30

  # 減少目標時長
  bun run merge-mp3 --input=... --target=28800 # 8 小時

  # 預覽分組結果
  bun run merge-mp3 --input=... --dry-run
  ```

  ### Q: "outOfMemory 錯誤"
  **A**: 並行度過高導致記憶體溢出
  ```bash
  # 減少並行度
  AUDIO_MERGE_MAX_CONCURRENCY=1 bun run merge-mp3 --input=...

  # 分批處理
  bun run merge-mp3 --input=part1 --output=merged1
  bun run merge-mp3 --input=part2 --output=merged2
  ```

  ## 5. MP4 轉換

  ### Q: "M4A 檔案無法播放"
  **A**: 可能是編碼或元資料問題
  ```bash
  # 驗證 M4A 檔案
  ffprobe output.m4a

  # 重新轉換
  bun scripts/mp3_to_mp4.ts --input=merged.mp3 --output=output.m4a

  # 嘗試不同的比特率
  MP4_BITRATE=192k bun scripts/mp3_to_mp4.ts --input=... --output=...
  ```

  ### Q: "元資料未嵌入"
  **A**: 檢查元資料 JSON 格式
  ```bash
  # 驗證 JSON 格式
  cat metadata.json | jq .

  # 檢查檔案名稱對應是否正確
  cat metadata.json | jq 'keys'

  # 使用 ffprobe 驗證嵌入
  ffprobe -v quiet -print_format json -show_format output.m4a | jq '.format.tags'
  ```

  ### Q: "轉換後檔案大小異常"
  **A**: 比特率設置可能不對
  ```bash
  # 預期大小計算
  # 大小 ≈ (時長(秒) * 比特率(kbps)) / 8

  # 查看實際比特率
  ffprobe -v error -select_streams a:0 -show_entries stream=bit_rate -of default=noprint_wrappers=1:nokey=1:noinvert_match=0 output.m4a

  # 使用特定比特率重新轉換
  MP4_BITRATE=256k bun scripts/mp3_to_mp4.ts --input=... --output=...
  ```

  ## 6. 性能問題

  ### Q: "轉換速度太慢"

  ```

  ### Q: "記憶體使用過高"
  **A**: 降低並行度或分批處理
  ```bash

  ```

  ```
